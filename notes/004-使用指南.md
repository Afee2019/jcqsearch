# jcqsearch 使用指南

## 一、工具简介

jcqsearch 是一款基于 PostgreSQL pg_trgm 的本地文件快速搜索工具，专为 macOS 设计，用于替代 Spotlight / Finder 搜索。

**核心特性：**

- 只扫描你关心的目录，不扫全盘
- 自动忽略 `node_modules`、`.git` 等无用目录
- 支持中文文件名模糊搜索（2 字及以上效果最佳）
- 支持关键词 + 文件类型 + 时间范围 + 文件大小的组合筛选
- 搜索速度极快（通常 < 10ms）

## 二、安装与初始化

### 2.1 前置条件

| 依赖 | 要求 |
|------|------|
| Go | ≥ 1.22 |
| PostgreSQL | ≥ 14，已安装 pg_trgm 扩展 |
| macOS locale | 系统已有 `zh_CN.UTF-8`（`locale -a | grep zh_CN` 验证） |

### 2.2 编译安装

```bash
cd /Users/shawn/dev/jcqsearch
go install
```

安装后二进制位于 `~/go/bin/jcqsearch`，确保 `$GOPATH/bin` 在 `$PATH` 中。

### 2.3 数据库初始化

首次使用需创建数据库并执行迁移脚本：

```bash
# 1. 创建数据库（支持中文排序和字符分类）
psql -U shawn -d postgres -c "
CREATE DATABASE jcqsearch
    WITH TEMPLATE = template0
    ENCODING = 'UTF8'
    LC_COLLATE = 'zh_CN.UTF-8'
    LC_CTYPE = 'zh_CN.UTF-8'
    TABLESPACE = pg_default;
"

# 2. 启用模糊搜索扩展
psql -U shawn -d jcqsearch -c "CREATE EXTENSION IF NOT EXISTS pg_trgm;"

# 3. 执行建表迁移
psql -U shawn -d jcqsearch -f internal/database/migrations/001_create_initial_tables.up.sql

# 4. 记录迁移版本
psql -U shawn -d jcqsearch -c "INSERT INTO schema_migrations (version, name) VALUES ('001', 'create_initial_tables');"
```

### 2.4 配置文件

jcqsearch 在以下位置查找 `config.yaml`（优先级从高到低）：

1. 当前工作目录 `./config.yaml`
2. 用户配置目录 `~/.config/jcqsearch/config.yaml`

也可通过环境变量覆盖，前缀为 `JCQSEARCH_`，层级用下划线分隔（如 `JCQSEARCH_DATABASE_PASSWORD`）。

**配置项说明：**

```yaml
database:
  host: localhost          # PostgreSQL 主机
  port: 5432               # PostgreSQL 端口
  dbname: jcqsearch        # 数据库名
  user: shawn              # 数据库用户
  password: ""             # 密码（本地 peer auth 可留空）
  sslmode: disable         # SSL 模式

scan:
  batch_size: 1000         # 批量写入条数（每攒够 N 条执行一次 UPSERT）
  concurrency: 4           # 并发扫描 goroutine 数（多路径并行扫描）

search:
  default_limit: 20        # find 命令默认返回条数
  similarity_threshold: 0.1  # pg_trgm 模糊匹配阈值（0~1，越低越宽松）
```

**阈值调整建议：**

| 阈值 | 效果 | 适用场景 |
|------|------|---------|
| 0.05 | 非常宽松，结果多但可能有无关项 | 记不清文件名，想大范围搜 |
| 0.1 | 默认值，平衡精准与召回 | 日常使用 |
| 0.2 | 较严格，结果少但更精准 | 文件名记得比较清楚 |
| 0.3 | 严格，仅高度相似 | 精确查找 |

## 三、快速上手

完成安装后，三步即可开始使用：

```bash
# 第 1 步：添加要扫描的目录
jcqsearch paths add ~/Desktop "桌面"
jcqsearch paths add ~/Documents "文档"

# 第 2 步：扫描建立索引
jcqsearch scan

# 第 3 步：搜索文件
jcqsearch find 报告
```

## 四、命令详解

### 4.1 jcqsearch scan — 扫描建立索引

扫描所有已配置的路径，将文件元信息（路径、名称、大小、修改时间等）写入数据库。

**基本用法：**

```bash
# 扫描所有已配置路径
jcqsearch scan

# 仅扫描指定路径（该路径必须已通过 paths add 添加）
jcqsearch scan ~/Desktop
```

**扫描行为：**

- 并发遍历各扫描路径（默认 4 个 goroutine）
- 自动跳过忽略规则中的目录和文件
- 每 1000 条批量写入数据库（UPSERT，冲突时更新）
- 扫描完成后自动清理数据库中已不存在的文件记录
- 遇到无权限的目录自动跳过并打印警告

**输出示例：**

```
扫描所有已配置路径:

  ✓ /Users/shawn/Desktop  2,802 个文件  (454ms)

✓ 扫描完成  共 2,802 个文件  耗时 457ms
```

**建议扫描频率：**

- 日常使用：每天扫描一次（可配合 cron 自动执行）
- 大量文件变动后：手动执行一次
- 搜索时发现文件找不到：执行一次 scan 更新索引

**cron 自动扫描示例：**

```bash
# 每天凌晨 2 点自动扫描（编辑 crontab: crontab -e）
0 2 * * * /Users/shawn/go/bin/jcqsearch scan 2>/dev/null
```

---

### 4.2 jcqsearch find — 搜索文件

核心命令，支持关键词模糊搜索和多维度组合筛选。

#### 4.2.1 基本搜索

```bash
# 模糊搜索文件名包含"报告"的文件
jcqsearch find 报告

# 多字关键词（空格连接，整体作为搜索词）
jcqsearch find 数据治理
```

#### 4.2.2 按文件类型筛选 (`-t` / `--type`)

```bash
# 搜索 PDF 文件
jcqsearch find 报告 -t pdf

# 搜索多种类型（逗号分隔）
jcqsearch find 报告 -t pdf,docx,pptx

# 不带关键词，仅按类型筛选
jcqsearch find -t mkv
```

扩展名不区分大小写，不需要加点号。

#### 4.2.3 按时间范围筛选 (`--after` / `--before` / `--recent`)

```bash
# 2024 年 1 月之后修改的文件
jcqsearch find 报告 --after 2024-01

# 指定时间区间
jcqsearch find 报告 --after 2024-01 --before 2024-06

# 精确到日期
jcqsearch find 报告 --after 2024-03-15

# 最近 7 天修改的文件
jcqsearch find --recent 7d

# 最近 24 小时
jcqsearch find --recent 24h

# 最近 30 分钟
jcqsearch find --recent 30m
```

**时间参数格式：**

| 格式 | 示例 | 说明 |
|------|------|------|
| `YYYY-MM` | `2024-01` | 精确到月 |
| `YYYY-MM-DD` | `2024-01-15` | 精确到日 |
| `YYYY-MM-DD HH:MM` | `2024-01-15 10:30` | 精确到分 |
| `Nd` | `7d` | 最近 N 天 |
| `Nh` | `24h` | 最近 N 小时 |
| `Nm` | `30m` | 最近 N 分钟 |

#### 4.2.4 按文件大小筛选 (`--larger` / `--smaller`)

```bash
# 大于 100MB 的文件
jcqsearch find --larger 100M

# 大于 1GB 的视频文件
jcqsearch find -t mkv,mp4,avi --larger 1G

# 10MB 到 100MB 之间的 PDF
jcqsearch find -t pdf --larger 10M --smaller 100M
```

**大小参数格式：**

| 后缀 | 含义 | 示例 |
|------|------|------|
| `K` | KiB (1024 字节) | `500K` |
| `M` | MiB (1024² 字节) | `100M` |
| `G` | GiB (1024³ 字节) | `1G` |
| 无后缀 | 字节 | `1048576` |

#### 4.2.5 按目录范围筛选 (`-d` / `--dir`)

```bash
# 仅在指定目录下搜索
jcqsearch find 报告 -d /Users/shawn/Desktop

# 搜索特定项目目录
jcqsearch find config -d /Users/shawn/dev/myproject
```

目录参数使用前缀匹配，会搜索该目录及其所有子目录。

#### 4.2.6 文件/目录筛选 (`--files-only` / `--dirs-only`)

```bash
# 仅搜索文件（不包含目录）
jcqsearch find 报告 --files-only

# 仅搜索目录
jcqsearch find project --dirs-only
```

#### 4.2.7 控制返回条数 (`-n` / `--limit`)

```bash
# 返回前 50 条结果（默认 20）
jcqsearch find 报告 -n 50

# 只看前 5 条
jcqsearch find 报告 -n 5
```

#### 4.2.8 快捷操作 (`-o` / `--reveal`)

```bash
# 搜索并用系统默认程序打开第 1 个结果
jcqsearch find 报告 -t pdf -o 1

# 搜索并在 Finder 中显示第 2 个结果
jcqsearch find 报告 -t pdf --reveal 2
```

`-o` 调用 macOS `open` 命令，根据文件类型自动选择关联程序（PDF → 预览，DOCX → Word，等）。

`--reveal` 调用 `open -R`，在 Finder 中高亮显示该文件。

#### 4.2.9 输出格式

```
  #  名称                                         大小         修改时间               路径

  1    数据治理平台产品白皮书V3.pdf                       2.1 MiB    2024-03-15 10:30   ~/dev/jcmdm/docs
  2    2024-数据治理方案.pdf                           856 KiB    2024-02-20 14:22   ~/Documents/projects
  3    数据治理培训材料.pdf                              5.4 MiB    2024-01-08 09:15   ~/Desktop

  共 3 条结果（耗时 7ms）
```

- 路径中 `/Users/shawn` 自动缩写为 `~`
- 搜索关键词在文件名中高亮显示（终端支持颜色时）
- 结果按相似度降序排列（有关键词时），无关键词时按修改时间降序

### 4.3 jcqsearch paths — 管理扫描路径

管理需要扫描和索引的目录列表。

```bash
# 列出所有扫描路径
jcqsearch paths list

# 添加路径（第二个参数为可选的备注）
jcqsearch paths add ~/Desktop "桌面"
jcqsearch paths add ~/Documents "文档"
jcqsearch paths add ~/dev "开发目录"

# 移除路径（同时自动清理该路径下的所有索引记录）
jcqsearch paths remove ~/Desktop
```

**paths list 输出示例：**

```
  ID  路径                                  备注           启用     深度         文件数        总大小

  1    /Users/shawn/Desktop                桌面           ✓      无限制        2,802      24 GiB
  2    /Users/shawn/Documents              文档           ✓      无限制        8,219      1.8 GiB
```

**注意事项：**

- 路径会自动转换为绝对路径
- 添加前会验证路径存在且为目录
- 重复添加同一路径会更新备注
- 移除路径时级联删除该路径下的所有文件索引（即时生效，无需重新扫描）
- 添加路径后需要执行 `jcqsearch scan` 才会建立索引

### 4.4 jcqsearch ignore — 管理忽略规则

控制扫描时需要跳过的目录和文件。

```bash
# 列出所有忽略规则
jcqsearch ignore list

# 添加规则：忽略名为 tmp 的目录
jcqsearch ignore add tmp dir

# 添加规则：忽略 .log 扩展名的文件
jcqsearch ignore add log ext

# 添加规则：忽略匹配 *.min.js 的文件
jcqsearch ignore add "*.min.js" glob

# 移除自定义规则
jcqsearch ignore remove tmp dir

# 移除预置规则（不会真正删除，而是禁用）
jcqsearch ignore remove vendor dir
# → 输出: 预置规则不可删除，已禁用: vendor (dir)
```

**规则类型说明：**

| 类型 | 参数 | 匹配方式 | 示例 |
|------|------|---------|------|
| `dir` | 目录名 | 遍历时遇到同名目录跳过整个子树 | `node_modules`, `.git`, `tmp` |
| `ext` | 扩展名 | 跳过该扩展名的文件（不含点号） | `pyc`, `log`, `tmp` |
| `glob` | 通配符 | 对文件名进行 glob 模式匹配 | `*.min.js`, `.DS_Store` |

**预置规则（24 条）：**

系统预置的忽略规则不可删除，仅可通过 `ignore remove` 禁用。预置规则涵盖：

- 版本控制目录：`.git`, `.svn`, `.hg`
- 包管理/依赖：`node_modules`, `vendor`, `.venv`, `__pycache__`
- 构建产物：`dist`, `build`, `target`, `.next`, `.nuxt`
- 缓存/临时：`.cache`, `.tox`, `.pytest_cache`
- macOS 系统文件：`.DS_Store`, `.Spotlight-V100`, `.Trashes`
- 编译产物扩展名：`pyc`, `pyo`, `o`, `so`, `dylib`, `class`

**添加规则后需要重新扫描** (`jcqsearch scan`) 才能生效。

## 五、实用搜索场景

### 5.1 日常文件查找

```bash
# 找一个名字记不全的 PDF
jcqsearch find 治理 -t pdf

# 找最近下载的文件
jcqsearch find --recent 1d

# 找大文件清理磁盘空间
jcqsearch find --larger 500M --files-only -n 50
```

### 5.2 按类型批量查找

```bash
# 找所有视频文件
jcqsearch find -t mp4,mkv,avi,flv,mov -n 50

# 找所有演示文稿
jcqsearch find -t pptx,ppt,key -n 50

# 找所有图片
jcqsearch find -t jpg,jpeg,png,gif,webp -n 50
```

### 5.3 时间维度查找

```bash
# 找上个月修改的文档
jcqsearch find -t pdf,docx,pptx --after 2026-02-01 --before 2026-03-01

# 找今天改过的文件
jcqsearch find --recent 1d --files-only

# 找过去一年没动过的大文件（先找大文件，再看时间）
jcqsearch find --larger 100M --before 2025-03 --files-only -n 50
```

### 5.4 定向目录搜索

```bash
# 仅在工作目录下搜索配置文件
jcqsearch find config -d ~/dev -t yaml,yml,json,toml

# 在文档目录找报告
jcqsearch find 报告 -d ~/Documents -t pdf
```

### 5.5 搜索 + 快捷操作

```bash
# 找到后直接打开
jcqsearch find 周报 -t pdf -o 1

# 找到后在 Finder 中定位
jcqsearch find 周报 -t pdf --reveal 1

# 先搜索看结果，再决定打开第几个
jcqsearch find 方案 -t pdf
# （查看结果后）
jcqsearch find 方案 -t pdf -o 3
```

## 六、中文搜索说明

jcqsearch 使用 PostgreSQL pg_trgm 扩展进行模糊搜索。pg_trgm 将字符串拆分为连续 3 字符片段（trigram）进行相似度计算。

### 6.1 搜索效果与关键词长度

| 关键词长度 | 效果 | 说明 |
|-----------|------|------|
| 1 个字 | 差 | trigram 不足，建议用 2 字以上 |
| 2 个字 | 良好 | 可以产生有效匹配 |
| 3 个字及以上 | 最佳 | trigram 充分，匹配精准 |

### 6.2 搜索原理

以搜索 `"数据"` 匹配 `"数据治理平台白皮书"` 为例：

- `"数据"` 的 trigram：`" 数据"`, `"数据 "` 等
- `"数据治理平台白皮书"` 的 trigram：`" 数据"`, `"数据治"`, `"据治理"`, ... 等
- 两者有共同的 trigram → 相似度 > 0 → 匹配成功

### 6.3 阈值调优

如果搜索结果不够理想，可以调整 `config.yaml` 中的 `similarity_threshold`：

```yaml
search:
  similarity_threshold: 0.05  # 降低阈值，获得更多结果
```

修改后立即生效，无需重新扫描。

## 七、管道与脚本集成

jcqsearch 的搜索结果输出到 stderr（状态信息）和 stdout（结果表格），可以与其他命令组合使用：

```bash
# 搜索结果传给 grep 二次过滤
jcqsearch find -t pdf -n 100 2>/dev/null | grep "2026"

# 配合 wc 统计数量
jcqsearch find -t py -n 1000 2>/dev/null | wc -l

# 定时扫描脚本
#!/bin/bash
jcqsearch scan 2>&1 | logger -t jcqsearch
```

## 八、常见问题

### Q1: 搜索没结果，但文件确实存在？

**原因：** 文件尚未被扫描入库。

**解决：**

```bash
# 确认文件所在目录已添加为扫描路径
jcqsearch paths list

# 如果没有，添加它
jcqsearch paths add /path/to/dir "备注"

# 重新扫描
jcqsearch scan
```

### Q2: 中文搜索结果太少？

**原因：** 默认相似度阈值可能偏高。

**解决：** 编辑 `config.yaml`，将 `similarity_threshold` 从 `0.1` 降到 `0.05`。

### Q3: 扫描速度慢？

**可能原因和解决方案：**

- 目录下文件太多 → 检查是否有大量 node_modules 等未被忽略的目录
- 添加忽略规则：`jcqsearch ignore add <目录名> dir`
- 调整并发数：修改 `config.yaml` 中 `scan.concurrency`

### Q4: 数据库连接失败？

**排查步骤：**

```bash
# 1. 确认 PostgreSQL 在运行
pg_isready

# 2. 确认数据库存在
psql -U shawn -d jcqsearch -c "SELECT 1;"

# 3. 检查配置文件
cat config.yaml
```

### Q5: 如何完全重置索引？

```bash
# 清空所有文件索引（保留路径和规则配置）
psql -U shawn -d jcqsearch -c "TRUNCATE files RESTART IDENTITY;"

# 重新扫描
jcqsearch scan
```

### Q6: 文件已删除但搜索仍能找到？

**原因：** 删除文件后未重新扫描。

**解决：** 执行 `jcqsearch scan`，扫描完成后会自动清理数据库中已不存在的文件记录。
